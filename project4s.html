Introduction:

This project analyzes pizza sales using SQL. The objective was to extract actionable insights for operational and strategic improvements, including peak sales time, best-selling products, revenue trends, and more.

Data Preparation:

  1. Data Source: Raw transactional dataset of a fictional pizza chain.
  2. Data Platform: MySQL for query execution.
  3. Data Cleaning: Null handling, column renaming, formatting dates.

SQL Queries and Analysis:

  1. Total Revenue: SELECT SUM(total_price) AS Total_Revenue FROM pizza_sales;
  2. Total Pizzas Sold: SELECT SUM(quantity) AS Total_Pizzas_Sold FROM pizza_sales;
  3. Average Order Value: SELECT SUM(total_price)/COUNT(DISTINCT order_id) AS Avg_Order_Value FROM pizza_sales;
  4. Best-Selling Pizza: SELECT pizza_name, SUM(quantity) AS Total_Sold FROM pizza_sales GROUP BY pizza_name ORDER BY Total_Sold DESC LIMIT 1;
  5. Hourly Demand: SELECT EXTRACT(HOUR FROM order_time) AS Hour, SUM(quantity) FROM pizza_sales GROUP BY Hour ORDER BY Hour;
  6. Peak Days: SELECT TO_CHAR(order_date, 'Day') AS Day, SUM(quantity) FROM pizza_sales GROUP BY Day;

KPIs:

  1. Total Revenue: $817,860.05
  2. Total Pizzas Sold: 49,574
  3. Average Order Value: $38.31
  4. Average Pizzas per Order: 2.72
